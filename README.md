# 企业人力资源数据分析库全流程构建与实战 | YGGL Database Project

## 一、项目简介
本项目是一个完整的数据库综合实战项目，模拟企业人力资源管理系统（YGGL），从零开始完成数据库的设计、数据维护及复杂数据分析。

项目完整覆盖数据库生命周期：
**环境搭建 → 数据建模 → 数据维护 → 复杂查询分析 → 数据视图封装**  
旨在系统性提升数据分析师在真实业务场景下对结构化数据的理解与操作能力。

---

## 二、技术栈
- 数据库：MySQL  
- 管理工具：Navicat  
- 语言：SQL（DDL / DML / DQL）  
- 核心能力：数据库设计、数据完整性、复杂查询、视图封装

---

## 三、数据库设计
### 核心数据表
- **Employees（员工表）**  
- **Departments（部门表）**  
- **Salary（薪资表）**

设计要点：
- 明确定义主键（PRIMARY KEY）与外键（FOREIGN KEY）
- 合理设置数据类型与是否允许空值
- 建立员工、部门、薪资之间的业务关联关系，保证数据一致性

---

## 四、项目内容与实践

### 1️⃣ 数据库环境与建模
- 安装并配置 MySQL 数据库
- 使用 SQL DDL 语句创建数据库与表结构
- 独立完成企业人力资源业务模型设计

### 2️⃣ 数据维护与完整性控制
- 使用 INSERT 批量插入模拟业务数据
- 使用 UPDATE / DELETE 完成条件更新与删除
- 通过外键约束保证数据完整性，避免脏数据产生

### 3️⃣ 核心查询与业务分析
- 完成 **50+ 条 SQL 查询任务**
- 覆盖内容包括：
  - 单表查询与条件筛选
  - 多表 JOIN 查询
  - 聚合函数（COUNT / SUM / AVG）
  - GROUP BY + HAVING
  - 子查询与嵌套查询
- 解决典型业务问题：
  - 各部门员工人数统计
  - 不同学历员工分布
  - 薪资对比分析（如高于某部门所有员工薪资）

### 4️⃣ 数据产品化：视图设计
- 将高频复杂查询封装为视图（VIEW）
- 简化分析流程，提高查询复用性
- 理解数据抽象与权限控制思想
